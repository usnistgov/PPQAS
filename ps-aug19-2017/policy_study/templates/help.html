User ID Creation Rules
A User ID must consist of only letters and numbers.

Recover Your User ID
Please keep your User ID safe. If you forget your user ID, we cannot recover it for you. We are only allowed to store a scrambled version to preserve your anonymity and protect your identity.

Unfortunately, if you cannot remember your User ID, you will need to create a new one for continued use of the application.

Since each policy review is associated with an individual User ID, any previous review effort continues to be associated with your former User ID and you will lose access to it.

Other
If you have other account related issues, contact the session administrator.

Browser Settings

NOTE: Only Firefox and Chrome are supported by this application.

Turning on JavaScript
Firefox
  1. In the address bar, type "about:config" (with no quotes), and click Enter.
  2. Click "I accept the risk!"
  3. In the search bar, search for "javascript.enabled" (with no quotes).
  4. If the value field is set to true for the result named "javascript.enabled", right click it and click "Toggle".
    5.  
  5. JavaScript is now enabled.
  6. To disable JavaScript, repeat these steps and toggle the value field for "javascript.enabled" to false.
Chrome
  1. Click the Chrome menu icon on the browser toolbar.
  2. Select Settings.
  3. On the "Settings" page, click the Show advanced settings... link.
  4. In the "Privacy" section, click Content settings...
    5.  
  5. Select Allow all sites to run JavaScript (recommended) in the "JavaScript" section.
    6.  
  6. Click Done.
Turning on Popups
Firefox
  1. Click the menu icon to open the Firefox menu.
  2. Click Options.
  3. Click Contents located on the left hand sidebar.
    4.  
  4. Uncheck the "Block pop-up windows" box.

Using this Tool to Identify Password Rules

Tips

  • Remember your User ID and keep it safe. If you forget it, you cannot recover it and you cannot go back to your previous answers. To review any policies, you'll have to create a new account and start all over again.
  • The tool tracks and saves your responses as you go. As long as you don't submit your responses, you can go back and change them, if you want.
  • Confused or concerned about a question or topic? Tell us in the comments at the end of each page.
  • To tell us about a more general concern, at the top of the page, click General Comment.
  • Some password policies do not present their rules clearly. At times the rules can be hard to understand. We want to know the rules you find stated in the policies you review.
  • The tool will show you 3 policies to review. Pick a policy to begin. You should probably work through each policy one at a time; but, you don’t have to submit your answers on one review before you start another.
  • For each policy you review, the tool will show you a series of questions about password rules. Your responses to the questions help identify each rule in the policy to us.

What's your job?

  • Identify each password rule in the policy.
    • Decide if there is a match between what each question in the application is asking and a stated rule in the policy.
    • When there is a match, additional response options open so you can tell us more about what the rule says.
    • Don't infer additional rules. Match questions only to rules actually stated in the policy.

      For example:
      If the policy contains a rule requiring a password change if the password is sent unencrypted,
      DO NOT INFER there is a rule that passwords must not be sent unencrypted.

  • What a rule means is sometimes difficult to uncover. Look at each rule carefully.
    1. Determine the main category {Communicating, Creating, Storing, Changing, Failing to Authenticate}
    2. Then look for a close match with the subtopics for that category.
    3. Once you find a subtopic match, determine if the rule states a requirement, prohibition, recommendation for or against.

    Use the following equivalencies:

    • Requirement        The password must…
    • Recommendation      The password should…
    • Prohibition          The password must not…
    • Recommendation against   The password should not…

  • Policies vary. Each policy you review will not have a rule for every topic we ask about, so don't worry. Just tell us that the category isn't mentioned and move on.
  • Ignore rules intended for accounts with special privileges or other authentication factors.

    We did not include questions about administrator account passwords or other authentication factors such as passphrases and SSH keys.


What else do you need to know?

  • You don’t have to answer every question, but we ask you to answer as much as you can. Write in comments if you want; we like details.
  • The browser's 'back' button may not work as expected on every page; the links and buttons within the application do.
  • At the end of a page, when you click Next, if you have missed a question or entered an invalid response, you’ll see . You may choose to answer the question, change your response, or move on.
  • When you get to the end of all of the questions, the tool will
    highlight
    the questions you haven’t answered.     It will suggest that you go back and complete them. At this point, you can either choose to go back or submit your answers.
  • Once you submit your answers, the tool marks your answers as complete and you cannot return to that policy.
